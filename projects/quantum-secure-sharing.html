Research Internship - NIT Agartala

Quantum-Safe Self-Destructing File Sharing

I built this as a secure file-sharing system where a document can be accessed once, verified through blockchain state, and protected with post-quantum cryptography.

Role
Flutter app, secure workflow, backend integration
Stack
Flutter, Flask, OQS, Kyber-512, AES-256-GCM, IPFS, Ethereum
Focus
Privacy-preserving access, self-destructing file flow, key handling

What I built

The system combines a mobile interface, a post-quantum backend, decentralized storage, and smart-contract verification. The main idea is simple: a sender shares a file securely, the receiver gets controlled one-time access, and the platform avoids keeping exposed files around after use.

Important parts

  • Used CRYSTALS-Kyber-512 for post-quantum key encapsulation.
  • Used AES-256-GCM for encrypted file handling.
  • Connected IPFS storage with Ethereum verification for access state.
  • Built Flutter flows for private-key handling and blockchain state checks.
  • Built Flask/OQS backend services for encapsulation and decapsulation.
